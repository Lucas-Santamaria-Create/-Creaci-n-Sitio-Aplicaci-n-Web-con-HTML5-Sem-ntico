Subconsultas en SQL

Las subconsultas en SQL son una herramienta poderosa que permite anidar consultas dentro de otras para realizar operaciones avanzadas. Aquí exploraremos cómo utilizarlas en diferentes contextos.

SELECT

Las subconsultas en SELECT permiten calcular valores para cada fila del resultado, utilizando datos de otras tablas o cálculos avanzados.

FROM

En FROM, las subconsultas actúan como "tablas temporales" que permiten realizar operaciones complejas de filtrado, agrupamiento o cálculo.

WHERE

Las subconsultas en WHERE ayudan a filtrar resultados basándose en condiciones derivadas de otras consultas.

HAVING

La cláusula HAVING usa subconsultas para aplicar condiciones a resultados de funciones de agregación como SUM, AVG, o COUNT.

JOIN

Las subconsultas en JOIN permiten unir datos de una tabla con los resultados de otra consulta para crear combinaciones complejas.

Funciones de Agregación Avanzadas

Las funciones de agregación avanzadas en SQL se utilizan para realizar cálculos sobre un conjunto de datos y devolver un único valor. Estas funciones son especialmente útiles para resumir y analizar grandes volúmenes de información.

Funciones Básicas

Cuenta el número de filas que cumplen con una condición específica.

-- Contar el número total de empleados en la tabla "empleados":
SELECT COUNT(*) AS TotalEmpleados
FROM empleados;

-- Contar solo empleados con un salario mayor a 5000:
SELECT COUNT(*) AS EmpleadosConSalarioAlto
FROM empleados
WHERE salario > 5000;

Suma los valores de una columna específica para todas las filas que cumplen con una condición.

-- Sumar el total de ventas realizadas:
SELECT SUM(monto_venta) AS TotalVentas
FROM ventas;

-- Sumar las ventas realizadas en un mes específico:
SELECT SUM(monto_venta) AS VentasEnero
FROM ventas
WHERE mes = 'Enero';

Calcula el promedio de los valores en una columna específica. Solo considera los valores no nulos.

-- Calcular el salario promedio de los empleados:
SELECT AVG(salario) AS SalarioPromedio
FROM empleados;

-- Calcular el promedio de calificaciones mayores a 60:
SELECT AVG(calificacion) AS PromedioAprobados
FROM estudiantes
WHERE calificacion >= 60;

Devuelve el valor máximo de una columna específica.

-- Encontrar el salario más alto:
SELECT MAX(salario) AS SalarioMaximo
FROM empleados;

-- Encontrar la venta más alta realizada en el año:
SELECT MAX(monto_venta) AS VentaMaxima
FROM ventas
WHERE anio = 2024;

Devuelve el valor mínimo de una columna específica.

-- Encontrar el salario más bajo:
SELECT MIN(salario) AS SalarioMinimo
FROM empleados;

-- Encontrar la calificación más baja en un examen:
SELECT MIN(calificacion) AS CalificacionMinima
FROM estudiantes;

Funciones Avanzadas

Las funciones avanzadas en SQL son esenciales para realizar cálculos y análisis más complejos. Estas herramientas permiten agrupar, filtrar y resumir datos de manera eficiente.

Utiliza junto con las funciones de agregación para agrupar resultados en función de una o más columnas. Cada grupo devuelve un valor de la función de agregación. 
  -- Calcular el salario promedio por departamento:
  SELECT departamento, AVG(salario) AS SalarioPromedio
  FROM empleados
  GROUP BY departamento;
  
  -- Contar el número de estudiantes por grado:
  SELECT grado, COUNT(*) AS NumeroEstudiantes
  FROM estudiantes
  GROUP BY grado;

Similar a WHERE, pero se usa para filtrar los resultados de un grupo después de aplicar las funciones de agregación. 
  -- Mostrar departamentos con salario promedio mayor a 5000:
  SELECT departamento, AVG(salario) AS SalarioPromedio
  FROM empleados
  GROUP BY departamento
  HAVING AVG(salario) > 5000;
  
  -- Mostrar grados con más de 10 estudiantes:
  SELECT grado, COUNT(*) AS NumeroEstudiantes
  FROM estudiantes
  GROUP BY grado
  HAVING COUNT(*) > 10;

Extensión de GROUP BY que genera subtotales en varias jerarquías de agrupación. 
  -- Generar subtotales por departamento y total general:
  SELECT departamento, SUM(salario) AS TotalSalario
  FROM empleados
  GROUP BY ROLLUP(departamento);

Similar a ROLLUP, pero genera subtotales para todas las combinaciones de las columnas especificadas. 
  -- Generar subtotales y totales cruzados por departamento y cargo:
  SELECT departamento, cargo, SUM(salario) AS TotalSalario
  FROM empleados
  GROUP BY CUBE(departamento, cargo);

Transacciones

Las transacciones SQL son un conjunto de operaciones que se ejecutan como una única unidad de trabajo. Esto significa que todas las operaciones dentro de una transacción deben completarse exitosamente para que se realicen los cambios en la base de datos. Si alguna de las operaciones falla, se pueden revertir todas las operaciones realizadas hasta ese momento, asegurando la integridad de los datos.

Comandos básicos de transacciones

Begin Transaction

Utilizar BEGIN TRANSACTION es esencial antes de realizar múltiples operaciones que deben considerarse como una unidad. Esto asegura que si ocurre un error en alguna de las operaciones, puedas revertir todas las operaciones realizadas desde que se inició la transacción.

Commit

Este comando finaliza la transacción y aplica todos los cambios realizados durante la transacción en la base de datos. Una vez que se ejecuta un COMMIT, los cambios son permanentes y no se pueden deshacer.

RollBack

Este comando deshace todos los cambios realizados en la transacción desde el BEGIN TRANSACTION. Se utiliza cuando hay un error en alguna de las operaciones y deseas revertir todos los cambios realizados hasta ese punto.
