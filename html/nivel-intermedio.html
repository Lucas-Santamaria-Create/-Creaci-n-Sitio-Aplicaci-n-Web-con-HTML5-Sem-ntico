Subconsultas

En SQL, una subconsulta es una consulta anidada dentro de otra consulta. Es una herramienta poderosa que permite utilizar el resultado de una consulta como entrada para otra. Las subconsultas se colocan normalmente entre paréntesis y pueden usarse en varias partes de una consulta principal, como en la cláusula SELECT, FROM, WHERE, HAVING, o JOIN

Select

"SELEC"T permiten calcular un valor que se utiliza en cada fila del resultado. Este valor puede venir de otra tabla o de un cálculo que no forma parte de las columnas principales de la consulta.

From

FROM se conocen como subconsultas de tabla o consultas derivadas. Estas subconsultas actúan como una "tabla temporal" que puedes usar en la consulta principal. Esto permite realizar operaciones complejas de filtrado, agrupamiento o cálculo que serían difíciles de hacer directamente en la consulta principal

Where

Las subconsultas en WHERE se utilizan para filtrar resultados en función de condiciones que se derivan de otra consulta. Esto permite realizar comparaciones más complejas o validar la existencia de registros relacionados en otras tablas.

Having

La cláusula HAVING se utiliza en SQL para filtrar los resultados de una consulta que involucra funciones de agregación (como SUM, AVG, COUNT, etc.). Al igual que en WHERE, puedes usar subconsultas en HAVING para aplicar condiciones basadas en resultados de otras consultas.

JOIN

Las subconsultas en JOIN se utilizan para unir resultados de una tabla con los resultados de otra consulta, que puede ser una subconsulta. Esto permite crear consultas complejas al combinar datos de diferentes tablas o conjuntos de resultados.

Funciones de Agregación Avanzadas

Las funciones de agregación avanzadas en SQL se utilizan para realizar cálculos sobre un conjunto de datos y devolver un único valor. Estas funciones son especialmente útiles para resumir y analizar grandes volúmenes de información.

  • Funciones Basicas:

    • COUNT(): Esta función cuenta el número de filas que cumplen con una condición específica. Puede contar todas las filas o solo aquellas que tienen un valor no nulo en una columna especificada.

    SUM(): Suma los valores de una columna específica para todas las filas que cumplen con una condición.

    AVG(): Calcula el promedio de los valores en una columna específica. Solo considera los valores no nulos.

    MAX(): Devuelve el valor máximo de una columna específica.

    MIN(): Devuelve el valor mínimo de una columna específica.

  • Funciones Avanzadas:

    • GROUP BY: Utiliza junto con las funciones de agregación para agrupar resultados en función de una o más columnas. Cada grupo devuelve un valor de la función de agregación.

    HAVING: Similar a WHERE, pero se usa para filtrar los resultados de un grupo después de aplicar las funciones de agregación.

    ROLLUP: Extensión de GROUP BY que genera subtotales en varias jerarquías de agrupación.

    CUBE: Similar a ROLLUP, pero genera subtotales para todas las combinaciones de las columnas especificadas.

Transacciones

Las transacciones SQL son un conjunto de operaciones que se ejecutan como una única unidad de trabajo. Esto significa que todas las operaciones dentro de una transacción deben completarse exitosamente para que se realicen los cambios en la base de datos. Si alguna de las operaciones falla, se pueden revertir todas las operaciones realizadas hasta ese momento, asegurando la integridad de los datos.

Propiedades de la transacciones

  • Atomicidad
  • Consistencia
  • Aislamiento
  • Durabilidad

Comando basicosde transacciones

  • Begin Transaction

    • Utilizar BEGIN TRANSACTION es esencial antes de realizar múltiples operaciones que deben considerarse como una unidad. Esto asegura que si ocurre un error en alguna de las operaciones, puedas revertir todas las operaciones realizadas desde que se inició la transacción.

  • Commit

    • Este comando finaliza la transacción y aplica todos los cambios realizados durante la transacción en la base de datos. Una vez que se ejecuta un COMMIT, los cambios son permanentes y no se pueden deshacer.

  • RollBack

    • Este comando deshace todos los cambios realizados en la transacción desde el BEGIN TRANSACTION. Se utiliza cuando hay un error en alguna de las operaciones y deseas revertir todos los cambios realizados hasta ese punto.
